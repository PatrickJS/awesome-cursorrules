# Great CTO Cursor Rules

Generated by [`npx great-cto adapt --platform cursor`](https://github.com/avelikiy/great_cto).

`great-cto` is an AI engineering management layer that:

- Auto-detects 1 of 25 project archetypes (fintech / commerce / healthcare / mlops / edtech / gov-public / insurance / etc.) and wires up matching compliance gates (PCI-DSS / GDPR / HIPAA / SOX / KYC-AML / COPPA / FedRAMP / NIST 800-53 / NAIC / Solvency II)
- Includes a 24-rule OWASP LLM Top 10 scanner (`great-cto scan` + SARIF for GitHub Security)
- Has cross-project memory (~/.great_cto/decisions.md ADR log queryable across all repos via MCP)
- Provides a 5-tool MCP server callable from Cursor's chat panel

## Quick start

```bash
# In your repo:
npx great-cto@latest init                      # bootstraps .great_cto/PROJECT.md
npx great-cto@latest adapt --platform cursor   # generates .cursorrules + .cursor/rules/*.mdc + AGENTS.md

# Add to .cursor/mcp.json for MCP integration:
{
  "mcpServers": {
    "great-cto": { "command": "npx", "args": ["-y", "great-cto@latest", "mcp"] }
  }
}
```

## Why this rule file

Most `.cursorrules` files lock in a stack convention. This one is **archetype-driven** — re-running `great-cto adapt` after editing `.great_cto/PROJECT.md` regenerates the rules to match your current archetype + compliance, so the file stays in sync with reality without manual edits.

The same source-of-truth (`.great_cto/PROJECT.md`) generates configs for Cursor, Claude Code, Codex CLI, Aider, and Continue — switch tools without rewriting conventions.
